Semantic HTML

Semantic HTML refers to html tags that convey the meaning or 'semantics' of the content within them. While it is possible to use the generic 'div' and 'span' for all of the elements on a page, that doesn't tell the web browser or anyone reading the code anything about whats going on on the page. Many users have imparements and must rely on a web reader or other accessibility feature in order to fully utilize the web. These features make use of semantic html to properly work. As a web developer it is important to use semantic html for these reasons.

<header>

The header tag is used for the header element. This is usually a banner image, the page title, and some meta information.

<nav>

The nav tag is used to hold the navigation links for the website.

<section>

A section is a thematic grouping of content, typically with a heading. An example would be a 'blog posts' section on a page. The whole page isn't about the blog posts, just that one section.

<article>

An article is a piece of indepentdent self-contained content. Each individual blog post within the 'blog posts' section would be an article.

<aside>

The aside tag is for content that is aside from the content it is placed in. A sidebar with ads or related posts, is an example of an aside.

<footer>

The footer is for the information at the bottom of the page. This generally will include some navigation, location information, coyright information, etc...

<figure>

The figure element is meant to wrap around an image element and a 'figcaption' element.

<img>

This is the tag for putting images into a web page.

<figcaption>

This tag is used for adding a caption to an image. It is used in conjunction with the 'figure' and 'img' tags.

<a>

The 'a' tag is for creating links.

<button>

This tag is for creating buttons. Many times the 'a' tag is used instead and styled to look like a button. The difference is 'a' or anchor tags are meant to be links to take users to different locations or pages. Buttons are meant for submitting information, not necessarily redirecting the user to a different web page.

<!-- comment text here... -->

This is the comment tag. it opens with <--! and closes with --> Everything inbetween those is considedered a comment and will only be visible to people who are reading the page code. It will not render on the web page.
